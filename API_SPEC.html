EcoFont Client API Spec

프론트엔드는 TTF 파일 하나만 보낸다. 서버는 TTF의 cmap Unicode coverage를 읽어 Cherokee 여부를 자동 판정하고, Cherokee면 eco 후보 TTF 20개와 평균 OCR 점수를 반환한다.

Header

Endpoint POST /v1/font-generation/ttf
Content-Type multipart/form-data
Authorization 없음
브라우저 FormData 사용 시 Content-Type을 직접 지정하지 않는다.

Request

{
  "font": File(.ttf 또는 TrueType glyf 기반 .otf)
}
font 필수. 사용자가 업로드한 원본 폰트 파일.
보내지 않는 값 script, method, candidate_count, spec JSON은 보내지 않는다.
서버 고정값 method=eco_research_guided, candidate_count=20, codepoint_set=cherokee_full, ocr_lang=chr

Response

성공 시 HTTP 200. 실제 TTF 파일은 JSON 안의 URL로 내려준다.

Download

전체 zip GET {API_BASE}{outputs.zip_url}
개별 TTF GET {API_BASE}{candidates[0].ttf_url}
Preview PNG GET {API_BASE}{candidates[0].preview_url}
응답의 *_url은 상대 경로다. 다운로드할 때 프론트에서 API_BASE를 앞에 붙인다.

Error

{
  "detail": "error message"
}

지원하지 않는 언어 폰트이면 detail이 객체로 내려올 수 있다.

{
  "detail": {
    "message": "uploaded font is not a supported Cherokee font",
    "detected_script": null,
    "script_counts": {
      "cherokee": 0,
      "hangul": 0
    },
    "supported_scripts": ["cherokee"]
  }
}
413업로드 파일이 50 MB를 초과함.
422빈 파일, 잘못된 폰트 형식, glyf 미지원, Cherokee 글자 없음.
500생성 중 서버 오류.
